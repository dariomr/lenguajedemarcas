HTML

Comienzo del documento HTML

Declaración tipo de documento <!DOCTYPE>

Se encuentra al principio del documento HTML y sirve como instrucción para el navegador sobre la versión de HTML en la que está escrito el documento HTML.
En HTML5 la etiqueta se declara así: <!DOCTYPE html>

<html>

El elemento <html> es el elemento raíz del documento HTML y se cierra al final del documento con la etiqueta </html>.
Se puede añadir un atributo que especifica el idioma así: <head lang ="es">

<head>

El elemento <head> provee información general acerca del documento, incluyendo el título, enlaces a scripts y hojas de estilos.
Se declara después de la etiqueta <html>

<title>

El elemento <title> especifica y da título al documento HTML.
Se declara dentro del elemento <head> y se cierra con </title>

<meta>

Proporciona metadatos (información) sobre el documento HTML aunque no se visualizan en la página web. Los metaelementos se usan generalmente para especificar la descripción de la página, las palabras clave, el autor del documento, la última modificación y otros metadatos. Se puede declarar así: <meta name="nombre" content="contenido">
La etiqueta meta va dentro del <head> y admite valores para name como: autor, descripcion... Por ejemplo: <meta name="autor" content="Darío Marín">
En HTML5 existe un nuevo atributo de la etiqueta <meta> para definir la codificación de carácteres que usa nuestro documento HTML. Ejemplo: <meta charset="UTF-8">

<link>

Se utiliza para especificar la relación entre el documento HTML y un recurso externo. Este elemento se usa para enlazar hojas de estilos con el documento HTML.
La etiqueta va dentro del <head> y se declara así: <link rel="stylesheet" type="text/css" href="hojaestilo.css">

Formato del documento

<body>

Se utiliza para representar el contenido de un documento HTML, que es lo único que visualizará el navegador. Sólo puede haber un elemento <body> en un documento y se cierra con su correspondiente etiqueta de cierre </body>

<header>

La etiqueta se utiliza para definir el encabezado de un documento o una sección, por lo que va dentro del <body> o de un elemento article o section. Puede contener algunos elementos de encabezado como un título, así como también un logo, un formulario de búsqueda, nombre de autor y otros componentes. Puede haber varios elementos <header> en el documento.
Se requiere una etiqueta inicial y una etiqueta final: <header> ... </header>

<footer>

Se utiliza para definir un pie de página del documento HTML. Contiene la información del pie de página (información del autor, información de derechos de autor, operadores, etc.). La etiqueta de pie de página se usa dentro de la etiqueta <body>.
Se requiere una etiqueta inicial y una etiqueta final: <footer> ... </footer>

<nav>

Representa una sección de una página cuyo propósito es proporcionar enlaces de navegación, ya sea dentro del documento actual o a otros documentos. Ejemplos comunes de secciones de navegación son menús, tablas de contenido e índices. Dentro de nav podemos insertar los enlaces que queremos poner para el menú y también pueden estar dentro de una lista para crear un menú lateral.
Ejemplo dentro de nav con lista: <ul> ... <li><a href="#">Inicio</a></li> ... </ul>

<aside>

Rpresenta una sección de una página que consiste en contenido que está indirectamente relacionado con el contenido principal del documento.
Estas secciones son a menudo representadas como barras laterales o como inserciones y contienen una explicación al margen como una definición de glosario, elementos relacionados indirectamente, como publicidad, la biografía del autor, o en aplicaciones web, la información de perfil o enlaces a blogs relacionados. Se requiere una etiqueta inicial y una etiqueta final: <aside> ... </aside>

<section>

Define la sección de documentos como capítulos, encabezados, pies de página o cualquier otra sección. La etiqueta <section> divide el contenido en secciones y subsecciones. Debería usarse para contenido que se agrupa de acuerdo con un tema, pero solo tiene sentido en el contexto del contenido que lo rodea.
Se requiere una etiqueta inicial y una etiqueta final: <section> ... </section>

<article>

Representa una composición auto-contenida en un documento, página, una aplicación o en el sitio, que se destina a distribuir de forma independiente o reutilizable, por ejemplo, en la sindicación. Podría ser un mensaje en un foro, un artículo de una revista o un periódico, una entrada de blog, un comentario de un usuario, un widget interactivo o gadget, o cualquier otro elemento independiente del contenido. Se debería utilizar cuando un contenido se puede sacar de una página por completo y todavía tiene sentido por sí mismo sin contenido circundante.
Se requiere una etiqueta inicial y una etiqueta final: <article> ... </article>

<adress>

Aporta información de contacto para su elemento <article> más cercano o el elemento principal <body>; en el último caso lo aplica a todo el documento.
Generalmente la información de contacto de una persona u organización se suele ubicar en el <footer> del documento HTML.

Agrupación de contenido

<h1 ... h6>

Los elementos de encabezado o heading implementan seis niveles de encabezado del documento, <h1> es el más importante y <h6> es el menos importante. Un elemento de encabezado describe brevemente el tema de la sección que presenta. La información de encabezado puede ser usada por los agentes usuarios, por ejemplo, para construir una tabla de contenidos para un documento automáticamente.
Por ejemplo: <h1>Este texto es del primer tipo</h1>

<p>

Elemento paragraph que sirve para definir un párrafo. Si se agregan varios espacios, el navegador los reduce a un solo espacio y lo mismo con los saltos de línea. Se requiere etiqueta de apertura y cierre.
Por ejemplo: <p>Esto es un párrafo</p>

<pre>

Sirve para representar texto preformateado. El texto en este elemento se muestra en una fuente fija (monoespaciada), no proporcional, exactamente como es mostrado en el archivo. Los espacios dentro de este elemento también son mostrados como están escritos.

<ol>

El elemento ordered list permite definir listas o viñetas ordenadas, bien con numeración o alfabéticamente.
Se pueden definir atributos como type="(1|a|A|i|I)" para dar diferentes estilos de numeración, compact para que el espacio entre los elementos de la lista sea reducido, start para indicar el número inicial de la lista ordenada y reversed para indicar si el orden va al revés. Dentro de la lista, los elementos se representan con la etiqueta <li>. Ejemplo: <ol> ... <li>punto 1</li> ... </ol>

<ul>

El elemento unordered list permite crear listas no ordenadas. Contiene uno o más elementos <li>. Admite atributos como: type="disc|square|circle" para definir el estilo de los elementos de la lista y compact para que el espacio entre los elementos de la lista sea reducido.

<hr>

En HTML5 representa un cambio de tema entre párrafos (por ejemplo, un cambio de escena en una historia, un cambio de tema en una sección). En versiones previas de HTML representaba una línea horizontal.
Admite atributos como size que indica la altura de la línea, width que indica la anchura (por defecto 100%). Se representa así: <hr/>

Elementos semánticos de texto

Resaltado

Dentro de un párrafo podemos escribir el texto en cursiva, negrita y subrayado.
Cursiva (italic): <i>Texto en cursiva</i> --> Texto en cursiva
Negrita (bold): <b>Texto en negrita</b> --> Texto en negrita
Subrayado (underlined): <u>Texto subrayado</u> --> Texto subrayado

También existen otras etiquetas de formato de texto como:
Emphasis: para marcar con énfasis las partes importantes de un texto. Ejemplo: <em>Texto énfasis</em> --> Texto énfasis
Strong: apropiado para marcar con especial énfasis las partes más importantes de un texto. Ejemplo: <strong>Texto strong</strong> --> Texto strong
Small: para representar comentarios laterales y letra pequeña, incluyendo derechos de autor y texto legal. Hace el tamaño del texto una talla más pequeña. Ejemplo: <small>Texto small</small> --> Texto small
S: muestra el texto tachado con una línea horizontal, texto que ya es irrelevante en el documento. Ejemplo: <s>Texto tachado</s> --> Texto tachado

Salto de línea

Break <br>: produce un salto de línea en el texto (retorno de carro). Es útil para escribir un poema o una dirección, donde la división de las líneas es significante.
Word break <wbr>: representa una posición dentro del texto donde el explorador puede opcionalmente saltar una línea , aunque sus reglas de salto de línea de otra manera no crearían un salto en esa posición.

Hiperenlace

La etiqueta <a> define un hipervínculo, que se utiliza para vincular de una página a otra. El atributo href indica la dirección de enlace.
Ejemplo: <a href="https://www.google.com">Este texto es un enlace a la página principal de Google</a> --> Este texto es un enlace a la página principal de Google
Además el enlace puede ser absoluto: http://sitio/pagina.html o relativo: pagina.html, subdirectorio/pagina.html, ../pagina.html
También puede haber un hiperenlace interno en el mismo documento poniendo un id al elemento que queremos enlazar. Ejemplo: <a href="#id del elemento">Enlace al elemento</a>
Enlace al principio del documento: href="#top"

Algunos atributos de la etiqueta a:
target="_blank": especifica dónde abrir el documento vinculado
download: especifica que el destino se descargará cuando un usuario haga clic en el hipervínculo

Imágenes y contenido encajado

Imágenes

<img>: representa una imagen en el documento.
Atributo principal <src> para indicar el origen (source) de la imagen.
Otros atributos para dar formato a la imgagen y características: <width>, para indicar el ancho de la imagen; <height>, para indicar el alto de la imagen; <alt>, texto alternativo que describe la imagen. Los usuarios lo verán; si la URL de la imagen es errónea.
Ejemplo: <img src=“imagen.png” width="300" height="300" alt=“Imagen del logo”>

Mapas de imagen

Sirve para crear una forma sobre una imagen que normalmente sirve de enlace a otro sitio al hacer click.
Ejemplo: <img usemap="#mapa">
El mapa se declara así: <map name="mapa"> y contiene el elemento <area> que contiene atributos:

shape: default, rect, circle, poly
coords: especifica las coordenadas del área del mapa. Los valores dependen del shape:
rect: x-izq, y-sup, x-dcha, i-inf
circle: x-centro, y-centro, radio
poly: x1, y1, x2, y2, ..., xN, yN
href: la url asociada
alt: describe el área o su vínculo, es necesario porque area no tiene contenido.
download: especifica que el destino se descargará cuando un usuario haga clic en el mapa de imagen
target: especifica dónde abrir el documento vinculado

Tablas

El elemento <table> representa datos tabulados en dos o más dimensiones.
Contiene elementos <tr>(filas) y <td>(columnas) Además. puede incluir el elemento <th>(encabezado).
Otros elementos dentro de la tabla: <thead> encabezado de tabla, <tbody> cuerpo de tabla, <tfoot> sumario de tabla y <caption> leyenda.

Algunos atributos de las tablas:
border="n": borde (en table)
rowspan="n": expansión en columna (en td y th)
colspan="n": expansión en fila (en td y th)

Formularios

Formularios

El elemento <form> representa una sección de un documento que contiene controles interactivos que permiten a un usuario enviar información a un servidor web.
El formlario se declara así: <form action="http://sitio.com/sub/adduser" method="post">
Atributos de form:
action: la URI de un programa que procesa la información enviada por medio del formulario
method: el método HTTP que el navegador usa para enviar el formulario. Puede ser:get, los datos del formulario son adjuntados a la URI del atributo action , con un '?' como separador, y la URI resultante es enviada al servidor o post, los datos del formulario son incluidos en el cuerpo del formulario y son enviados al servidor.
enctype: cuando el valor del atributo method es post, tipo de mediatype que es usado para enviar el formulario. Posibles valores: application/x-www-form-urlencoded, el valor por defecto si un atributo no está especificado y multipart/form-data si se está usando el elemento <input> con el atributo type ajustado a "file".
accept-charset: una lista de codificación de caracteres que el servidor acepta.

El elemento <label> representa una etiqueta para un elemento en una interfaz de usuario.
La etiqueta se declara así: <label for="nombre">Nombre: </label>
Atributo de label: for, el ID del elemento de formulario etiquetable relacionado en el mismo documento que el elemento label. El primer elemento en el documento con tal ID que coincida con el dispuesto en el atributo for será el control etiquetado para este elemento.

El elemento <input> está relacionado con label y se usa para crear controles interactivos para formularios basados en la web, que reciban datos del usuario.
Atributos de input:
type: El tipo de control a mostrar. Su valor predeterminado es text, si no se especifica este atributo. Los posibles valores son:

type="text": campo de texto de línea simple.
type="submit": botón que envía el formulario.
type="number": control para introducir un valor númerico. Admite atributos como min, max, value (número real) y step="n"
type="checkbox": casilla de selección o verificación. Se debe usar el atributo value para definir el valor que se enviará por este elemento, checked para indicar si el elemento está seleccionado y name.
type="radio": botón de opción. Se debe usar el atributo value para definir el valor que se enviará por este elemento, checked para indicar si el elemento está seleccionado de forma predeterminada. y name.
También existen otros atributos como: password, search, tel, url, email, range, hidden, image(con su src), file accept, button, reset, date, month, week, time, color...

name: para indicar el nombre del input
id: enlaza con el for de un label
value: el valor inicial del control. Este atributo es opcional, excepto cuando el atributo type es radio o checkbox.
required: el usuario debe llenar el control antes de enviarlo al formulario.
size: el tamaño inicial del control. Este valor es en píxeles, a menos que el atributo type sea text o password, en cuyo caso será el número entero de caracteres. 

Ejemplo:
<form action=http://somesite.com/prog/adduser method=post>
    <p>
        <label for=nombre>Nombre: </label>
            <input type=text id=nombre name=nombre><br>
        <label for=apellido>Apellido: </label>
            <input type=text id=apellido name=apellido><br>
        <label><input type=email name=email>E-mail: </label><br>
            <input type=radio name=sexo value=hombre>hombre<br>
            <input type=radio name=sexo value=mujer>mujer<br>
            <input type=submit value=Enviar> <input type=reset>
    </p>
</form>

Menú

Para declarar el menú usamos el elemento <select> que puede tener los atributos name, size y multiple
Después especificamos los elementos <option> del menú que tienen los atributos value y selected

Ejemplo: 
<select name=menum size=3 multiple>
    <option selected value=uno>uno</option>
    <option value=dos>dos</option>
    <option value=tres>tres</option>
    <option selected value=cuatro>cuatro</option>
    <option value=cinco>cinco</option>
</select>

El elemento <optgroup> se utiliza para crear un conjunto de opciones similares y tiene el atributo label. 

Ejemplo:
<select name=menuo>
    <option selected value=uno>uno</option>
    <optgroup label=dos>
    <option value=dos_a>dos - a</option>
    <option value=dos_b>dos - b</option>
    </optgroup>
    <option selected value=tres>tres</option>
</select>

Editor de texto

El elemento <textarea> se utiliza para representar un control para edición muti-línea de texto plano.
Admite los atributos name, cols y rows para dar formato al cuadro.

Ejemplo:
<textarea name=texto_largo cols=20 rows=6>
    Este texto esta; contenido en el cuadro del
    editor.
</textarea>

Lista de valores posibles

El elemento <datalist> contiene un conjunto de elementos <option> que representan los valores disponibles para otros controles.
El elemento datalist admite el atributo id y el elemento option contiene el atributo value

Ejemplo:
<input name=sexo list=sexos>
<datalist id=sexos>
    <option value=mujer>
    <option value=hombre>
</datalist>

Botón

El elemento <button> representa un elemento cliqueable de tipo botón que puede ser utilizado en formularios o en cualquier parte de la página que necesite un botón estándar y simple de aplicar.
Tiene atributos como name, value y type(submit, button o reset).

Ejemplo:
<button name=submit value=submit
    type=submit>Enviar<IMG src=enviar.gif
    alt=Enviar>
</button>
<button name=reset type=reset>Limpiar<IMG
    src=borrar.gif alt=Borrar>
</button>
Multimedia y contenidos encajados

Plugins

El elemento <embed> representa un punto de integración para una aplicación externa o de contenido interactivo
Atributos que tiene:
src: la dirección URL del recurso que se está incrustado.
type: el tipo MIME que se va a usar para la selección del plug-in para crear una instancia.
Multipurpose Internet Mail Extensions o MIME son una serie de convenciones o especificaciones dirigidas al intercambio a través de Internet de todo tipo de archivos de forma transparente para el usuario.
parámetro del plugin: por ejemplo quality.

Ejemplo: <embed src=juego.swf quality=high>

Marcos

El elemento <iframe> representa un contexto de navegación anidado, el cual permite incrustrar otra página HTML en la página actual.
Tiene atributos como src(source), srcdoc, sandbox(restricciones adicionales al contenido en el marco),allowfullscreen, width, height...

Ejemplo:

<iframe id="inlineFrameExample"
    title="Inline Frame Example"
    width="300"
    height="200"
    src="https://www.openstreetmap.org/export/...">
</iframe>

Audio y vídeo

Los elementos <audio> y <video> sirven para insertar elementos multimedia.
Admiten atributos como: src, width(vídeo) y height(vídeo)

Ejemplo de audio:

<audio src=musica.mp3>
    El navegador no soporta audio
</audio>

También admiten otros atributos como controls, autoplay, muted, loop, preload="none|auto|metadata" (variaciones en la forma en que el navegador carga el vídeo o audio) y type (por ejemplo: type="video/mp4").

Ejemplo de vídeo:

<video controls>
    <source src=video.mp4 type="video/mp4">
    <source src=video.ogv type='"video/ogg"codecs="theora, vorbis"'>
    El navegador no soporta video
</video>

Se puede dar valores al vídeo como por ejemplo el rango de reproducción:
#t=[inicio][,fin] en segundos o horas:minutos:segundos

Existe el elemento <track> que va dentro de vídeo y se utiliza para especificar subtítulos, archivos de subtítulos u otros archivos que contienen texto, que deberían estar visibles cuando se reproducen los medios.
Admite los atributos src, kind(subtitles, captions, descriptions, chapters, metadata), srclang, label, default...

Ejemplo de track:

<video src=video.mp4>
    <track kind=subtitles src=video.en.vtt 
    srclang=en label=English>
    <track kind=captions src=video.en.hoh.vtt 
    srclang=en label="English for deaf">
    <track kind=subtitles src=video.fr.vtt 
    srclang=fr label=Français>
    <track kind=subtitles src=video.es.vtt
    srclang=es label=Español default>
        El navegador no soporta video
</video>

CSS

Sintaxis CSS

Introducción

La sintaxis de todos los elementos en CSS es así: 

selector { propiedad: valor; }

Ejemplo:

body { background-color: yellow;
            color: blue;
            margin-left: 50%; }

Colores

Nombre: black, white, grey, yellow, orange, blue, red, green, purple...
Modelo RGB: Hexadecimal: #RGB (0-F) o #RRGGBB (00-FF), rgb/rgba: rgb(rojo,verde,azul) - (0-255) o rgb(r%, v%, a%) - (0-100)
Es importante elegir bien el contraste entre los colores y tener en cuenta el daltonismo. Para ello hay herramientas que comprueban la validez de las combinaciones de colores.

Clases

Para crear diferentes estilos de un mismo elemento podemos utilizar clases.

Ejemplo CSS:

p { color: red; }
p.morado { color: purple; }
                        
El contenido del html sería este:

<p>Este texto es rojo</p>
<p class="morado">Este texto es morado </p>

Unidades

Se usan porcentajes u otras medidas como:
Relativas: em, ex
Aboslutas: px, pt, pc, in, cm, mm
Ángulos: deg, grad, rad
Duración: s, ms
Frecuencia: Hz, kHz
Resolución: dpi, dpdm, dppx

Fuentes

El atributo para definir el tipo de fuente es font-family.

Existen familias genéricas de fuentes que son las más utilizadas:
serif: Times New Roman, Garamond
sans-serif: Arial, Verdana, Helvética
monospace: Courier, Courier New
fantasy: western

El atributo font-style nos permite cambiar el estilo de la fuente de ese elemento con los atributos normal, italic y obloque.
El atributo font-variant nos permite dar los valores como normal y small-caps a la fuente.
El atributo font-weight admite valores como bolder, ligher:100-900; normal:400 o bold:700
El atributo font-height que sirve para determinal la altura de la fuente y admite los atributos: normal o una medida
El atributo font-size que determina el tamaño de letra de la fuente, con atributos absolutos(small, medium, large...), atributos relativos (larger, smaller) o medidas

Texto

Podemos definir el sangrado con el atributo text-indent y la alineación del texto con text-align (left, right, center, justify)
El atributo letter-spacing y word-spacing se usan para modificar los espacios
El atributo text-transformation se utiliza para dar un formato al texto con valores como capitalize, uppercase, lowercase, none
El atributo text-decoration también da formato al texto con valores como none, underline, overline, line-through, blink
El atributo vertical-align se usa para la alineación vertical del texto con valores como baseline, middle, sub, super, text-top, text-bottom, top, bottom

Listas

Atributo list-style-type para listas que pueden ser:
Ordenadas: lower-roman, upper-roman, decimal, decimal-leading-zero, upper-alpha, lower-alpha, hebrew, armenian, ...
No ordenadas: disc, circle, square, ...
Atributos list-style-image, list-style-position (outside,inside) y list-style

Fondo de objeto

Atributos para el fondo de un elemento:
background-color, background-image, background-repeat(repeat-x, repeat-y, no-repeat, none), background-attachment(scroll, fixed), background-position(porcentaje, medida, top, bottom, center...)

Modelo de caja

Margen

Atributos para definir el margen como: margin, margin-top, margin-right, margin-bottom, margin-left

Ejemplo:

body { margin-top: 150px;
            magin-right: 50px;
            margin-bottom: 20px;
            margin-left: 100px; }

Para el padding funciona igual.

Bordes

Atributos para los bordes: border-width(valor, thin, medium, thick), border-style(none, dotted, dashed, solid, double, groove, ridge, inset, outset, hidden), border-color
